## Visão Geral

O Kivo Pay abstrai a complexidade da rede Stellar para todos os módulos do ecossistema. Ele é responsável por:
- Criação e gestão de carteiras Stellar (custodial e non-custodial).
- Roteamento de pagamentos (Path Payments).
- Integração com âncoras SEP-24 para on/off-ramp (fiat ↔ crypto).

**Status:** Core engine funcional na landing page. Integração real com Stellar Testnet ativa no ContractEase (ancoragem de hashes).

### Ancoragem de Hash (Usado pelo ContractEase)

```typescript
// Gerar hash do documento
const hash = await crypto.subtle.digest('SHA-256', fileBuffer);
const hashHex = Array.from(new Uint8Array(hash))
  .map(b => b.toString(16).padStart(2, '0'))
  .join('');

// Criar transação Stellar com hash no Memo
const transaction = new StellarSdk.TransactionBuilder(account, {
  fee: StellarSdk.BASE_FEE,
  networkPassphrase: StellarSdk.Networks.TESTNET
})
  .addOperation(StellarSdk.Operation.payment({
    destination: destinationAddress,
    asset: StellarSdk.Asset.native(),
    amount: '0.0000001' // Transação mínima
  }))
  .addMemo(StellarSdk.Memo.text(hashHex.substring(0, 28)))
  .setTimeout(30)
  .build();
```

## Fluxos de Pagamento

### 1. Pagamento Simples (P2P)
```
Sender → Kivo Pay → Stellar Network → Receiver
```

### 2. Path Payment (Conversão automática)
```
Sender (USDC) → Stellar DEX → Receiver (BRL-token)
```
Usa `PathPaymentStrictReceive` para garantir que o destinatário receba o valor exato.

### 3. Batch Payment (Payouts)
```
Admin → CSV upload → Kivo Pay agrupa em Transaction Envelope → Stellar processa lote
```

## Carteiras

### Custodial (Gerenciada pelo backend)
- Chave privada armazenada em segredo no backend (Edge Function).
- Usada para operações automatizadas (ancoragem, payouts).
- **Nunca expor seed no frontend.**

### Non-Custodial (Freighter)
- Usuário gerencia sua própria carteira via extensão Freighter.
- Usado para assinatura de contratos e pagamentos manuais.
- Frontend usa `@stellar/freighter-api` para interação.

---

## Regras para a IA

1. **Sempre usar Testnet** durante desenvolvimento.
2. **Nunca expor chaves privadas** no código frontend.
3. **Memo limit:** 28 caracteres para `Memo.text()` na Stellar.
4. **Transação mínima:** `0.0000001 XLM` para operações de ancoragem (sem valor financeiro).
5. **Timeout:** Sempre definir `setTimeout(30)` em transações para evitar transações órfãs.
