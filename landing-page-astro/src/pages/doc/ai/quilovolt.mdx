# ⚡ Specs: QuiloVolt

Integração entre Hardware IoT (Totens EV) e a rede Stellar.

---

## Comunicação IoT

- **Protocolo:** OCPP (Open Charge Point Protocol).
- **Bridge:** Gateway Node.js que traduz mensagens OCPP para chamadas Stellar.

---

## Fluxo de Pagamento (Micropayments)

Para evitar centenas de pequenas transações on-chain, o QuiloVolt pode usar **Soroban State Channels** ou **Claimable Balances** com retenção.

### Cenário A: Pré-autorização
1. Usuário autoriza `50 USDC` (Hold).
2. Carregamento inicia.
3. No fim, o valor exato é capturado e o restante devolvido.

### Cenário B: Canal de Pagamento (Soroban)
1. Abre canal com depósito inicial.
2. Incrementos de `0.1 USDC` por minuto/kWh off-chain.
3. Fecha canal e liquida saldo final on-chain.

---

## Regras para a IA
1. **Offline Mode:** Se o totem perder conexão, deve permitir o início da recarga se o usuário já tiver uma sessão pré-autorizada válida em cache local.
2. **Segurança:** O ID do Totem deve estar vinculado a uma chave pública autorizada na tabela `ev_chargers`.
3. **Hardware:** A interação NFC usa o padrão NDEF para passar a URI de pagamento Stellar para a wallet do celular do usuário.
