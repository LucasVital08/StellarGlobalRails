# 🏥 Specs: Saúde 360

Gerenciamento soberano de dados clínicos.

---

## Criptografia (Client-Side)

O SGR nunca vê os dados em texto claro.
- **Key Derivation:** PBKDF2 a partir da semente da carteira Stellar do usuário.
- **Data Encryption:** AES-256-GCM.
- **Key Exchange:** RSA-4096 para compartilhamento com terceiros.

---

## Modelo de Dados (Planejado)

### `data_access_logs`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` | PK |
| `owner_id` | `uuid` | Paciente |
| `requester_id` | `uuid` | Empresa/Pesquisador |
| `data_hash` | `text` | Referência ao documento cifrado |
| `consent_tx_hash` | `text` | Tx Stellar comprovando o consentimento |
| `payment_tx_hash` | `text` | Tx Stellar do pagamento (USDC) |

---

## Regras para a IA
1. **Zero Knowledge:** O servidor não deve processar chaves privadas de decriptografia. Tudo ocorre no dispositivo do paciente ou via Soroban Smart Contract de custódia de chaves.
2. **Revogação:** Implementar sistema de rotação de chaves. Ao revogar, o documento é cifrado com uma nova chave inacessível ao antigo pesquisador.
3. **DID:** Usar o padrão `did:stellar:{account_id}` para identificação do paciente.
