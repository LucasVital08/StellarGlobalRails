# 📄 Specs: ContractEase B2B

> **Para IA:** Este documento é a fonte única da verdade para o módulo ContractEase. Leia-o inteiramente antes de implementar qualquer feature relacionada.

---

## Visão Geral do Módulo

O ContractEase é um SPA React (Vite + TypeScript) localizado em `apps/contractease/`. Ele gerencia o ciclo de vida de contratos B2B com multi-tenancy, assinatura digital e ancoragem de hash na Stellar Testnet.

**Projeto Supabase ID:** `fpmhcjkadgebcexzkayz`

---

## Fluxos Críticos

### Fluxo de Assinatura
```
1. Admin cria contrato → status = 'draft'
2. Adiciona signatários (contract_parties) → status = 'pending'
3. Signatário recebe e-mail com Magic Link
4. Abre link → página pública de assinatura
5. Solicita OTP (código de 6 dígitos enviado por e-mail)
6. Valida OTP → Coleta assinatura (draw/type/freighter)
7. Registra: signed_at, ip_address, user_agent, geolocation
8. Quando todos assinaram → gera SHA-256 do documento
9. Envia hash para Stellar Testnet (Transaction Memo)
10. Grava stellar_tx_hash no contrato → status = 'completed'
```

### Fluxo de Multi-tenancy
```
1. Usuário faz login (Magic Link)
2. Sistema busca organization_members para user_id
3. Se possui 1+ orgs → exibe seletor de workspace
4. Workspace selecionado → organization_id setado no contexto
5. Todas as queries filtram por organization_id (RLS)
```

## Regras para a IA

1. **Nunca hardcodar `organization_id`** — sempre obter do store/contexto do usuário.
2. **Sempre verificar RLS** — se criar uma nova tabela, ativar RLS e criar políticas.
3. **Supabase Storage** — contratos são armazenados no bucket `contracts` do Supabase Storage.
4. **Stellar** — usar Testnet para desenvolvimento. Nunca expor seeds no frontend.
