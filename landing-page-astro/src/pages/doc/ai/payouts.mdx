# 💸 Specs: Stellar Payouts

O módulo de Payouts gerencia o envio de ativos para múltiplos destinatários em uma única operação atômica.

---

## Mecanismo: Batching (Transaction Envelopes)

Diferente de enviar uma transação por vez, o Payouts agrupa até **100 operações** dentro de uma única transação Stellar (limite da rede).

### Exemplo de Construção de Lote (IA)

```typescript
const transactionBuilder = new StellarSdk.TransactionBuilder(sourceAccount, {
  fee: StellarSdk.BASE_FEE,
  networkPassphrase: StellarSdk.Networks.TESTNET
});

// Adicionando múltiplos pagamentos ao mesmo envelope
payoutsList.forEach(payout => {
  transactionBuilder.addOperation(StellarSdk.Operation.payment({
    destination: payout.address,
    asset: StellarSdk.Asset.native(),
    amount: payout.amount
  }));
});

const transaction = transactionBuilder.build();
```

## Modelo de Dados (Planejado)

### `payout_batches`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` | PK |
| `organization_id` | `uuid` | FK → organizations |
| `status` | `text` | `'processing'` \| `'completed'` \| `'failed'` |
| `total_amount` | `numeric` | Soma de todos os pagamentos |
| `stellar_tx_hash` | `text?` | Hash da transação única do lote |

### `payout_items`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` | PK |
| `batch_id` | `uuid` | FK → payout_batches |
| `recipient_address` | `text` | G-Address |
| `amount` | `numeric` | Valor individual |
| `status` | `text` | Status individual (redundante, mas útil para logs) |

## Regras para a IA
1. **Limite de Lote:** Se a lista de pagamentos for maior que 100, a IA deve dividir em múltiplas transações.
2. **Fee Bump:** Para pagamentos B2B, a organização paga as taxas de rede (Source Account paga a fee).
3. **CSV Import:** O componente de upload deve validar endereços Stellar e formatos de número antes de submeter ao backend.
4. **Memo:** Usar `Memo.id` (ID do Lote) para reconciliação bancária/contábil.
