# Data Models

Kivo Pay uses Supabase PostgreSQL as its primary database. All access from the Go backend goes through `pgx v5` with type-safe queries generated by `sqlc`. This document covers the full schema, generated Go types, example queries, and Row Level Security policies.

---

## Schema Overview

```
auth.users (Supabase managed)
    │
    ├── devices (one owner → many devices)
    │       │
    │       └── payments (from_device_id / to_device_id)
    │               │
    │               └── payment_conditions (one payment → many conditions)
    │
    ├── webhooks (one owner → many webhooks)
    │       │
    │       └── webhook_deliveries (one webhook → many deliveries)
    │
    └── api_keys (one owner → many API keys)
```

## Table: `devices`

```sql
-- migrations/001_create_devices.sql

CREATE TABLE devices (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name                TEXT NOT NULL,
    owner_id            UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    api_key_hash        TEXT NOT NULL UNIQUE,   -- SHA-256 of the raw API key
    stellar_public_key  TEXT NOT NULL UNIQUE,   -- Stellar G... address
    encrypted_secret    TEXT NOT NULL,          -- AES-256-GCM encrypted Stellar secret
    status              TEXT NOT NULL DEFAULT 'active'
                            CHECK (status IN ('active', 'suspended', 'decommissioned')),
    metadata            JSONB DEFAULT '{}',     -- device-specific key/value data
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_devices_owner_id ON devices(owner_id);
CREATE INDEX idx_devices_api_key_hash ON devices(api_key_hash);
CREATE INDEX idx_devices_stellar_public_key ON devices(stellar_public_key);

-- RLS
ALTER TABLE devices ENABLE ROW LEVEL SECURITY;

CREATE POLICY devices_owner_select ON devices
    FOR SELECT USING (owner_id = auth.uid());

CREATE POLICY devices_owner_insert ON devices
    FOR INSERT WITH CHECK (owner_id = auth.uid());

CREATE POLICY devices_owner_update ON devices
    FOR UPDATE USING (owner_id = auth.uid());
```

Generated Go struct (via sqlc):

## Table: `payments`

```sql
-- migrations/002_create_payments.sql

CREATE TABLE payments (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    from_device_id      UUID NOT NULL REFERENCES devices(id),
    to_device_id        UUID NOT NULL REFERENCES devices(id),
    amount              NUMERIC(20, 7) NOT NULL CHECK (amount > 0),
    asset_code          TEXT NOT NULL DEFAULT 'USDC',   -- USDC or XLM
    asset_issuer        TEXT,                            -- null for XLM
    condition_type      TEXT NOT NULL DEFAULT 'none'
                            CHECK (condition_type IN (
                                'none',
                                'energy_kwh',
                                'time_elapsed',
                                'service_complete',
                                'custom'
                            )),
    condition_value     TEXT,                            -- e.g. "10.5" for 10.5 kWh
    status              TEXT NOT NULL DEFAULT 'pending'
                            CHECK (status IN (
                                'pending',
                                'processing',
                                'confirmed',
                                'failed',
                                'expired',
                                'refunded'
                            )),
    stellar_hash        TEXT UNIQUE,                     -- Stellar transaction hash
    memo                TEXT,                            -- optional payment memo
    timeout_at          TIMESTAMPTZ,                     -- null = no timeout
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    confirmed_at        TIMESTAMPTZ,
    failed_reason       TEXT
);

CREATE INDEX idx_payments_from_device ON payments(from_device_id);
CREATE INDEX idx_payments_to_device ON payments(to_device_id);
CREATE INDEX idx_payments_status ON payments(status);
CREATE INDEX idx_payments_stellar_hash ON payments(stellar_hash);

-- RLS: both parties can see the payment
ALTER TABLE payments ENABLE ROW LEVEL SECURITY;

CREATE POLICY payments_participant_select ON payments
    FOR SELECT USING (
        from_device_id IN (SELECT id FROM devices WHERE owner_id = auth.uid())
        OR
        to_device_id   IN (SELECT id FROM devices WHERE owner_id = auth.uid())
    );

CREATE POLICY payments_initiator_insert ON payments
    FOR INSERT WITH CHECK (
        from_device_id IN (SELECT id FROM devices WHERE owner_id = auth.uid())
    );
```

Generated Go struct:

```go

    CreatedAt      time.Time        `json:"created_at"`
    ConfirmedAt    *time.Time       `json:"confirmed_at,omitempty"`
    FailedReason   *string          `json:"failed_reason,omitempty"`
}
```

## Table: `payment_conditions`

Stores the proof that a condition was met — e.g. a signed meter reading from an EV charger.

```sql
CREATE TABLE payment_conditions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    payment_id      UUID NOT NULL REFERENCES payments(id) ON DELETE CASCADE,
    condition_key   TEXT NOT NULL,       -- e.g. "energy_kwh"
    expected_value  TEXT NOT NULL,       -- e.g. "10.5"
    actual_value    TEXT,                -- value reported by the device
    proof_data      JSONB DEFAULT '{}',  -- signed reading, oracle response, etc.
    met_at          TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_payment_conditions_payment_id ON payment_conditions(payment_id);
```

## Table: `webhooks`

```sql
CREATE TABLE webhooks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id    UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    url         TEXT NOT NULL,
    events      TEXT[] NOT NULL DEFAULT '{}',   -- e.g. ARRAY['payment.confirmed','payment.failed']
    secret      TEXT NOT NULL,                   -- HMAC-SHA256 signing secret (hashed)
    active      BOOLEAN NOT NULL DEFAULT TRUE,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE webhooks ENABLE ROW LEVEL SECURITY;
CREATE POLICY webhooks_owner ON webhooks USING (owner_id = auth.uid());
```

## Table: `webhook_deliveries`

```sql
CREATE TABLE webhook_deliveries (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    webhook_id      UUID NOT NULL REFERENCES webhooks(id) ON DELETE CASCADE,
    payment_id      UUID REFERENCES payments(id),
    event           TEXT NOT NULL,
    payload         JSONB NOT NULL,
    status          TEXT NOT NULL DEFAULT 'pending'
                        CHECK (status IN ('pending', 'delivered', 'failed')),
    attempts        INT NOT NULL DEFAULT 0,
    response_code   INT,
    next_retry_at   TIMESTAMPTZ,
    delivered_at    TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_webhook_deliveries_status ON webhook_deliveries(status);
CREATE INDEX idx_webhook_deliveries_next_retry ON webhook_deliveries(next_retry_at)
    WHERE status = 'failed';
```

## Table: `api_keys`

Dashboard users can create multiple named API keys with specific scopes.

```sql
CREATE TABLE api_keys (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id    UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    name        TEXT NOT NULL,
    key_hash    TEXT NOT NULL UNIQUE,      -- SHA-256 of raw key
    scopes      TEXT[] NOT NULL DEFAULT '{}',
                -- e.g. ARRAY['devices:write','payments:read','webhooks:write']
    last_used_at TIMESTAMPTZ,
    expires_at   TIMESTAMPTZ,              -- null = never expires
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE api_keys ENABLE ROW LEVEL SECURITY;
CREATE POLICY api_keys_owner ON api_keys USING (owner_id = auth.uid());
```

## sqlc Configuration

```yaml
# sqlc.yaml
version: "2"
sql:
  - engine: "postgresql"
    queries: "queries/"
    schema: "migrations/"
    gen:
      go:
        package: "db"
        out: "internal/db"
        sql_package: "pgx/v5"
        emit_json_tags: true
        emit_pointers_for_null_types: true
        overrides:
          - db_type: "uuid"
            go_type: "github.com/google/uuid.UUID"
          - db_type: "numeric"
            go_type: "github.com/shopspring/decimal.Decimal"
          - db_type: "timestamptz"
            go_type: "time.Time"
```

## sqlc Query Examples

### queries/payments.sql

```sql
-- name: CreatePayment :one
INSERT INTO payments (
    from_device_id, to_device_id, amount, asset_code,
    condition_type, condition_value, timeout_at, memo
) VALUES ($1, $2, $3, $4, $5, $6, $7, $8)
RETURNING *;

-- name: GetPaymentByID :one
SELECT * FROM payments WHERE id = $1;

-- name: ListPaymentsByDevice :many
SELECT * FROM payments
WHERE from_device_id = $1 OR to_device_id = $1
ORDER BY created_at DESC
LIMIT $2 OFFSET $3;

-- name: UpdatePaymentStatus :one
UPDATE payments
SET status = $2, stellar_hash = $3, confirmed_at = $4, failed_reason = $5
WHERE id = $1
RETURNING *;

-- name: GetExpiredPendingPayments :many
SELECT * FROM payments
WHERE status = 'pending'
  AND timeout_at IS NOT NULL
  AND timeout_at < NOW();
```

### queries/devices.sql

```sql
-- name: CreateDevice :one
INSERT INTO devices (name, owner_id, api_key_hash, stellar_public_key, encrypted_secret)
VALUES ($1, $2, $3, $4, $5)
RETURNING *;

-- name: GetDeviceByAPIKeyHash :one
SELECT * FROM devices WHERE api_key_hash = $1 AND status = 'active';

-- name: GetDeviceByID :one
SELECT * FROM devices WHERE id = $1;

-- name: ListDevicesByOwner :many
SELECT id, name, stellar_public_key, status, created_at, metadata
FROM devices
WHERE owner_id = $1
ORDER BY created_at DESC;
```

## RLS Policy Summary

| Table | Policy | Rule |
|---|---|---|
| `devices` | SELECT / INSERT / UPDATE | `owner_id = auth.uid()` |
| `payments` | SELECT | either party's device belongs to `auth.uid()` |
| `payments` | INSERT | `from_device_id` device belongs to `auth.uid()` |
| `webhooks` | ALL | `owner_id = auth.uid()` |
| `api_keys` | ALL | `owner_id = auth.uid()` |

The Go backend (using the service role key for worker processes) bypasses RLS when needed for internal operations such as payment status updates by the Stellar monitor goroutine. Application-layer auth handlers always use the user's JWT context, which enforces RLS automatically.
