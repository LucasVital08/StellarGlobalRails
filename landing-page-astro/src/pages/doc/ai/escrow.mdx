# 🛡️ Specs: Stellar Escrow

Implementação de custódia segura on-chain.

---

## Mecanismo: Claimable Balances

Para o MVP, usamos Claimable Balances da Stellar (nativos) em vez de Soroban complexo para reduzir latência e custo.

```typescript
// Criar Escrow (Comprador deposita)
const op = StellarSdk.Operation.createClaimableBalance({
  asset: USDC_ASSET,
  amount: '100.00',
  claimants: [
    new StellarSdk.Claimant(sellerAddress, StellarSdk.Claimant.predicateNot(StellarSdk.Claimant.predicateBeforeAbsoluteTime(unlockTime))),
    new StellarSdk.Claimant(buyerAddress, StellarSdk.Claimant.predicateBeforeAbsoluteTime(unlockTime)) // Reembolso se expirar
  ]
});
```

## Modelo de Dados (Planejado)

### `escrows`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` | PK |
| `balance_id` | `text` | ID do Claimable Balance na Stellar |
| `buyer_id` | `uuid` | FK → profiles |
| `seller_id` | `uuid` | FK → profiles |
| `status` | `text` | `'held'` \| `'claimed'` \| `'refunded'` \| `'dispute'` |
| `unlock_at` | `timestamptz` | Data máxima para liberação |

---

## Regras para a IA
1. **Time-locks:** Sempre definir um `unlock_at` (ex: 7 dias) para que o dinheiro não fique preso para sempre se o vendedor sumir.
2. **Taxas:** O comprador paga a taxa de criação do balanço.
3. **Notificação:** O vendedor deve ser notificado por e-mail/push no exato momento em que o `createClaimableBalance` for confirmado na rede.
