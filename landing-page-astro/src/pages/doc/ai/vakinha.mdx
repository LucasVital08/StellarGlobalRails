# 🤝 Specs: Vakinha Global

O Vakinha Global utiliza a rede Stellar para garantir transparência absoluta em arrecadações de fundos.

---

## Modelo de Dados (Planejado)

### `campaigns`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` (PK) | Auto-gerado |
| `organization_id` | `uuid` | FK → `organizations.id` |
| `title` | `text` | Nome da campanha |
| `goal_amount` | `numeric` | Meta em USDC |
| `collected_amount` | `numeric` | Valor já arrecadado |
| `stellar_escrow_address` | `text` | Carteira que detém os fundos |
| `status` | `text` | `'active'` \| `'completed'` \| `'paused'` |

### `donations`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` (PK) | Auto-gerado |
| `campaign_id` | `uuid` | FK → `campaigns.id` |
| `donor_address` | `text` | G-Address do doador |
| `amount` | `numeric` | Valor doado |
| `stellar_tx_hash` | `text` | Hash da transação na rede |
| `message` | `text?` | Mensagem de apoio |

---

## Fluxo Técnico

1. **Criação:** Uma nova conta Stellar (Escrow) é gerada para cada campanha.
2. **Recebimento:** O doador envia USDC para o `stellar_escrow_address`.
3. **Auditoria:** O frontend lê as transações diretamente da Horizon API para exibir o feed de doações em tempo real.
4. **Repasse:** O admin da campanha solicita o saque. O Kivo Pay processa o off-ramp para a conta bancária/Pix cadastrada.

## Regras para a IA
1. **Transparência:** O feed de doações deve ser lido da blockchain, não apenas do banco de dados, para garantir veracidade.
2. **Taxas:** Informar ao doador que a taxa de rede é paga pelo emissor (frações de centavo).
3. **Memo:** Usar `Memo.id` (ID da campanha) em todas as doações.
