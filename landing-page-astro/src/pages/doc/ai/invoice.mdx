# 🧾 Specs: Stellar Invoice

O sistema de Invoice vincula documentos de cobrança a transações Stellar.

---

## Modelo de Dados (Planejado)

### `invoices`
| Coluna | Tipo | Notas |
|---|---|---|
| `id` | `uuid` (PK) | Auto-gerado |
| `organization_id` | `uuid` | FK → `organizations.id` |
| `customer_email` | `text` | E-mail do pagador |
| `amount` | `numeric` | Valor em USDC |
| `status` | `text` | `'draft'` \| `'sent'` \| `'paid'` \| `'overdue'` |
| `memo_id` | `text` | Text Memo único para reconciliação |
| `stellar_tx_hash` | `text?` | Registro da liquidação |
| `due_date` | `timestamptz` | Prazo de vencimento |

---

## Lógica de Reconciliação

O serviço `InvoiceWatcher` monitora a conta de recebimento da organização via stream de transações:

```typescript
server.payments()
  .forAccount(orgWallet)
  .cursor('now')
  .stream({
    onmessage: (payment) => {
      if (payment.memo === invoice.memo_id) {
        markInvoiceAsPaid(invoice.id, payment.transaction_hash);
      }
    }
  });
```

## Regras para a IA
1. **Unicidade do Memo:** Cada invoice aberta deve ter um `memo_id` único para aquela carteira de recebimento.
2. **Webhooks:** Disparar webhook para o sistema do cliente assim que o status mudar para `'paid'`.
3. **PDF:** O PDF da fatura deve conter o QR Code com o `Stellar URI (SEP-7)` pré-configurado com o valor e o memo.
