# 🛡️ Specs: Onyx Risk

O Onyx Risk é a camada de segurança e compliance do ecossistema. Ele analisa endereços Stellar e fornece um score de risco baseado em histórico de transações e grafos de vizinhança.

---

## APIs e Integrações

### Horizon API (Sourcing)
- **Endpoint:** `GET /accounts/{account_id}/transactions`
- **Uso:** Extração massiva de histórico para análise de padrões.

### Graph Engine (Core)
- **Algoritmo:** PageRank Reverso + DBScan (Clusterização).
- **Lógica:** Identificar "taint" (contaminação) de carteiras conhecidas por atividades ilícitas.
- **Vizinhança:** Análise de até 3 graus de separação.

---

## Modelo de Dados (Planejado)

### `risk_scores`
Cache de scores para evitar re-análise constante.

| Coluna | Tipo | Notas |
|---|---|---|
| `account_id` | `text` (PK) | G-Address Stellar |
| `score` | `int` | 0 (Seguro) a 100 (Alto Risco) |
| `flags` | `text[]` | Motivos do score (ex: `['mixer_usage', 'high_volume']`) |
| `last_analyzed` | `timestamptz` | Timestamp da última varredura |

---

## Implementação sugerida para IA

Para implementar verificações de risco em novos fluxos, utilize o seguinte padrão:

```typescript
async function validateReceiverRisk(address: string) {
  const { data: risk } = await supabase
    .from('risk_scores')
    .select('*')
    .eq('account_id', address)
    .single();

  if (risk && risk.score > 80) {
    throw new Error('Transação bloqueada: Destinatário de alto risco.');
  }
}
```

## Regras para a IA
1. **Cache First:** Sempre consulte `risk_scores` antes de disparar uma análise pesada via Graph Engine.
2. **Latência:** O endpoint de consulta de score deve responder em menos de 100ms.
3. **Privacidade:** Não armazene dados pessoais, apenas a chave pública e metadados de rede.
