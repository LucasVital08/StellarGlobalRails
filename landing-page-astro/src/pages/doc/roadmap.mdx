# 🗓️ Roadmap & Visão Pós-Hackathon

O Stellar Global Rails não nasceu para morrer no hackathon. Ele é um ecossistema de longo prazo, com fases claras de maturação.

---

## Fase 1 — Hackathon (Atual)
**Objetivo:** Provar o conceito e demonstrar a viabilidade técnica do ecossistema integrado.

| Entrega | Status |
|---|---|
| Landing Page com módulos interativos | ✅ Concluído |
| ContractEase MVP (contratos + assinatura + ancoragem) | ✅ Concluído |
| SocialPay Simulador funcional | ✅ Concluído |
| Documento de Produto (ADR/RT) | ✅ Concluído |
| Portal de documentação `/doc` | ✅ Concluído |

## Fase 2 — Q3/2026 (Pós-Hackathon)
**Objetivo:** Validação de mercado real e primeiros usuários.

- Lançamento do **ContractEase** em beta fechado para 5–10 PMEs brasileiras.
- Backend funcional do **SocialPay**: mapeamento real de `@handles` para carteiras Stellar.
- Integração do **Kivo Pay** com pelo menos uma âncora SEP-24 operacional (on/off-ramp BRL).
- Primeiros testes de **FamilyBridge** com remessas EUA → Brasil.

## Fase 3 — Q4/2026
**Objetivo:** Escala e monetização.

- API pública do Kivo Pay para integradores terceiros.
- Dashboard administrativo white-label para empresas B2B.
- **Onyx Risk** em beta: scoring de carteiras e análise de risco em tempo real.
- Início do compliance regulatório para operações de câmbio.

## Fase 4 — 2027
**Objetivo:** Licenciamento e expansão geográfica.

- Licenciamento institucional do **Onyx Risk** e **ContractEase** para bancos e fintechs.
- Expansão para corredores de remessa América Latina ↔ Europa.
- SDK mobile para integração nativa em apps parceiros.

---

> Este roadmap é um documento vivo e será atualizado conforme novos aprendizados e validações de mercado surgirem.
