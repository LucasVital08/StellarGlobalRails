# Stellar Integration

SocialPay settles all payments on the Stellar network. This document covers the custodial wallet model, XLM and USDC payment flows, trustlines, Horizon API usage, Etherfuse ramp integration, and error handling.

---

## Custodial Wallet Architecture

Every SocialPay user has exactly one Stellar account, created at signup. The platform holds the secret key encrypted in Supabase Vault. From the user's perspective, they never interact with a wallet — they just send to @handles.

### Wallet Initialization

When a user signs up, a Stellar keypair is generated server-side and the account is optionally funded:

```typescript
// lib/stellar/wallet.ts
import { Keypair, Networks, Server } from '@stellar/stellar-sdk';

export async function createStellarAccount(userId: string, network: 'testnet' | 'mainnet') {
  const keypair = Keypair.random();

  // Save to Vault (see auth-security.mdx for Vault implementation)
  await saveToVault(userId, keypair.secret());

  if (network === 'testnet') {
    // Fund via Friendbot (testnet only)
    await fetch(`https://friendbot.stellar.org?addr=${keypair.publicKey()}`);
  }

  return keypair.publicKey();
}
```

## XLM Payment Flow

XLM (Stellar Lumens) payments are the simplest path: no trustline required, no path finding, just a direct payment operation.

```typescript
// lib/stellar/payments.ts
import {
  Keypair,
  Networks,
  Operation,
  Server,
  TransactionBuilder,
  Asset,
  Memo,
} from '@stellar/stellar-sdk';

const HORIZON_URL = process.env.STELLAR_NETWORK === 'mainnet'
  ? 'https://horizon.stellar.org'
  : 'https://horizon-testnet.stellar.org';

const NETWORK_PASSPHRASE = process.env.STELLAR_NETWORK === 'mainnet'
  ? Networks.PUBLIC
  : Networks.TESTNET;

const server = new Server(HORIZON_URL);

export interface PaymentParams {
  sourceSecret: string;
  destination: string;  // Stellar public key (G...)
  amount: string;       // e.g. "50.0000000"
  asset: 'XLM' | 'USDC';
  memo?: string;
}

export async function sendXlmPayment(params: PaymentParams): Promise<{ hash: string }> {
  const sourceKeypair = Keypair.fromSecret(params.sourceSecret);
  const sourceAccount = await server.loadAccount(sourceKeypair.publicKey());

  const transaction = new TransactionBuilder(sourceAccount, {
    fee: await server.fetchBaseFee(),
    networkPassphrase: NETWORK_PASSPHRASE,
  })
    .addOperation(
      Operation.payment({
        destination: params.destination,
        asset: Asset.native(),
        amount: params.amount,
      })
    )
    .addMemo(params.memo ? Memo.text(params.memo.slice(0, 28)) : Memo.none())
    .setTimeout(30)
    .build();

  transaction.sign(sourceKeypair);
  const result = await server.submitTransaction(transaction);

  return { hash: result.hash };
}
```

## USDC Trustline Setup

Before a Stellar account can hold USDC, it must establish a trustline to the USDC issuer. This is a one-time operation per account.

### Why Trustlines Exist

Stellar requires explicit opt-in to hold any non-native asset. This prevents spam assets from being sent to accounts without consent and ensures the account holder acknowledges the issuer.

### USDC Issuer Addresses

| Network | Issuer |
|---|---|
| Mainnet | `GA5ZSEJYB37JRC5AVCIA5MOP4RHTM335X2KGX3IHOJAPP5RE34K4KZVN` |
| Testnet | `GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5` |

### Trustline Operation

```typescript
// lib/stellar/trustline.ts
export async function establishUsdcTrustline(sourceSecret: string): Promise<{ hash: string }> {
  const USDC_ISSUER = process.env.STELLAR_NETWORK === 'mainnet'
    ? 'GA5ZSEJYB37JRC5AVCIA5MOP4RHTM335X2KGX3IHOJAPP5RE34K4KZVN'
    : 'GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5';

  const usdcAsset = new Asset('USDC', USDC_ISSUER);
  const sourceKeypair = Keypair.fromSecret(sourceSecret);
  const sourceAccount = await server.loadAccount(sourceKeypair.publicKey());

  const transaction = new TransactionBuilder(sourceAccount, {
    fee: await server.fetchBaseFee(),
    networkPassphrase: NETWORK_PASSPHRASE,
  })
    .addOperation(
      Operation.changeTrust({
        asset: usdcAsset,
        limit: '1000000', // max USDC the account will hold
      })
    )
    .setTimeout(30)
    .build();

  transaction.sign(sourceKeypair);
  const result = await server.submitTransaction(transaction);

  return { hash: result.hash };
}
```

SocialPay automatically establishes the USDC trustline for all new user accounts during onboarding, so users never encounter the trustline requirement manually.

```typescript
// lib/stellar/balance.ts
export interface StellarBalance {
  asset_code: string;
  asset_issuer: string | null;
  balance: string;
}

export async function getWalletBalances(publicKey: string): Promise<StellarBalance[]> {
  const account = await server.loadAccount(publicKey);

  return account.balances.map((b: any) => ({
    asset_code: b.asset_type === 'native' ? 'XLM' : b.asset_code,
    asset_issuer: b.asset_type === 'native' ? null : b.asset_issuer,
    balance: b.balance,
  }));
}
```

```typescript
export async function fundTestnetWallet(publicKey: string): Promise<void> {
  if (process.env.STELLAR_NETWORK !== 'testnet') {
    throw new Error('Friendbot is only available on testnet');
  }

  const response = await fetch(
    `https://friendbot.stellar.org?addr=${encodeURIComponent(publicKey)}`
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`Friendbot error: ${error.detail || response.statusText}`);
  }
}
```

## Etherfuse Ramp Integration

Etherfuse is a SEP-24 compatible anchor providing BRL and MXN on/off-ramps for Stellar. SocialPay integrates with Etherfuse's REST API.

```typescript
// lib/etherfuse.ts
const ETHERFUSE_BASE = 'https://api.etherfuse.com';
const ETHERFUSE_API_KEY = process.env.ETHERFUSE_API_KEY!;

export async function getEtherfuseQuote(params: {
  type: 'on_ramp' | 'off_ramp';
  fiat_currency: 'BRL' | 'MXN';
  fiat_amount: number;
}) {
  const response = await fetch(`${ETHERFUSE_BASE}/v1/quote`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${ETHERFUSE_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(params),
  });

  if (!response.ok) throw new Error(`Etherfuse quote error: ${response.statusText}`);
  return response.json();
}

export async function createEtherfuseOrder(params: {
  type: 'on_ramp' | 'off_ramp';
  fiat_currency: 'BRL' | 'MXN';
  fiat_amount: number;
  stellar_account: string;  // destination for USDC credit
}) {
  const response = await fetch(`${ETHERFUSE_BASE}/v1/order`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${ETHERFUSE_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(params),
  });

  if (!response.ok) throw new Error(`Etherfuse order error: ${response.statusText}`);
  return response.json();
}
```

## Error Handling

All Stellar operations must handle a defined set of error conditions:

```typescript
// lib/stellar/errors.ts
export function handleStellarError(error: unknown): never {
  if (error instanceof Error) {
    const msg = error.message;

    if (msg.includes('op_no_destination')) {
      throw new Error('DESTINATION_NOT_FOUND: Recipient Stellar account does not exist. They may need to activate their wallet first.');
    }
    if (msg.includes('op_underfunded')) {
      throw new Error('INSUFFICIENT_BALANCE: Sender wallet does not have enough XLM or USDC to complete this payment.');
    }
    if (msg.includes('op_no_trust')) {
      throw new Error('TRUSTLINE_MISSING: Recipient has not established a USDC trustline. They must set up their wallet before receiving USDC.');
    }
    if (msg.includes('tx_insufficient_fee')) {
      throw new Error('STELLAR_ERROR: Transaction fee too low. The network is congested — please retry.');
    }
    if (msg.includes('timeout')) {
      throw new Error('STELLAR_ERROR: Transaction timed out. The network may be congested — check Stellar Expert to see if the transaction was submitted.');
    }
  }

  throw new Error(`STELLAR_ERROR: ${String(error)}`);
}
```

| Error Code | Cause | Resolution |
|---|---|---|
| `op_no_destination` | Recipient account not activated | Fund recipient account (testnet: Friendbot; mainnet: send XLM first) |
| `op_underfunded` | Insufficient XLM or USDC | User must top up via Etherfuse ramp or receive funds |
| `op_no_trust` | Recipient has no USDC trustline | Call `POST /api/wallet/trustline` for the recipient |
| `tx_insufficient_fee` | Network congestion | Retry with higher fee; use `fetchBaseFee()` dynamically |
| `tx_bad_seq` | Sequence number mismatch | Reload account and retry; avoid parallel submissions |
